# Contributing to LearnLab Bridge

Thank you for your interest in contributing to the LearnLab Bridge project! This tool is designed to empower teachers and students with a unified Data Science workspace.

## How to Contribute

### 1. Reporting Bugs
- Search existing issues to see if the bug has already been reported.
- If not, create a new issue with a clear title and description.
- Include steps to reproduce the bug and details about your environment.

### 2. Feature Requests
- Check if the feature has already been requested.
- Open a new issue describing the feature and why it would be useful for teachers/students.

### 3. Submitting Changes
- Fork the repository.
- Create a new branch for your changes (`git checkout -b feature/awesome-feature`).
- Make your changes and ensure they follow the project's coding standards.
- Run tests and linting (`npm run lint`, `npm run build`).
- Commit your changes with a descriptive message.
- Push to your fork and submit a Pull Request.

## Development Workflow
- **Issue First:** We prefer starting with an issue to discuss the design before implementation.
- **Code Review:** All pull requests require at least one review from a maintainer.
- **Testing:** New features should include relevant tests.

## Coding Standards
- Use TypeScript for type safety.
- Follow the existing project structure and naming conventions.
- Keep components small and focused.
- Ensure all UI changes are responsive and accessible.

## Team Roles
- **Maintainers:** Core developers responsible for the project's direction and merging PRs.
- **Contributors:** Community members who submit improvements and fixes.
- **Educators:** Subject matter experts who help define requirements for teaching.

Join us in building the future of Data Science education!
